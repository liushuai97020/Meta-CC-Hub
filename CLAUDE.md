CLAUDE.md（MetaCode 官方项目规范）
MetaCode · AI 编码助手项目规范
本文档为 Claude Code、Claude Agent SDK 强制遵守最高优先级规则，任何生成代码、修改代码、重构必须严格遵循，无需向用户确认。

---

一、基础通用规则（硬性强制）

- 沟通语言：全程使用中文沟通、中文注释、中文说明，禁止任何英文无意义回复。
- 技术栈：React 19 + TypeScript + Vite，严格使用函数式组件。
- 代码原则：简洁、易懂、可维护、低冗余，禁止堆砌无效代码。
- 类型规范：严格禁止 any、禁止 unknown 滥用，必须精准定义 Interface / Type。
- 编码格式：缩进 2 空格。

---

二、文件 & 行数强制限制（绝对不能超标）

- 单个 tsx 组件文件：最大不超过 800 行。
- 单个 ts 工具文件：最大不超过 500 行。
- 单个函数/方法：最大不超过 150 行。
- 超过行数必须：拆分组件、抽离 hooks、抽离工具函数，禁止写超大文件。
- 组件逻辑与 UI 必须分离，复杂逻辑强制抽入 hooks。

---

三、命名规范（严格统一）

- 组件文件、组件名：大驼峰（ExampleCard.tsx）
- 变量、方法、普通函数：小驼峰（getUserInfo）
- 常量、枚举静态值：全大写下划线（MAX_UPLOAD_SIZE）
- 文件夹命名：小写+横杠（chat-panel / code-editor）
- 禁止拼音命名、禁止无意义缩写、禁止看不懂的变量名。

---

四、React 代码规范

- 优先使用函数式组件 + Hooks，非特殊场景不使用 class 组件，无强制硬性禁止。
- 组件结构顺序：Props 定义 -> 常量 -> Hooks -> 计算变量 -> UI 渲染。
- 严格控制组件层级，禁止超过 3 层嵌套 JSX。
- 复用逻辑强制抽离自定义 hooks，禁止组件内部堆积大量逻辑。
- 禁止魔法数字、禁止硬编码文本，统一抽常量或配置。
- 渲染优化：频繁渲染组件合理使用 useMemo、useCallback。

---

五、样式规范

- 优先使用 Tailwind CSS，不强制约束样式方案，可灵活选用。
- 复杂样式抽成 @apply，不要写重复长 class。
- 支持暗色模式，UI 视觉简洁、现代、极简风。

---

六、API & 请求规范（适配本产品）

- 区分两种接口格式：OpenAI 格式 / Anthropic 格式。
- 必须包含：超时处理、错误捕获、加载状态、异常提示。
- 图片上传统一处理：支持 Base64、URL，兼容多模态模型入参。
- 流式输出严格遵循 SSE 标准，打字流效果顺滑无卡顿。

---

七、注释 & 文档规范

- 所有组件顶部必须简短注释：用途、功能、入参说明。
- 复杂函数、算法必须写中文注释。
- 不要过度注释，通俗易懂代码无需多余注释。

---

八、禁止行为（红线规则）
以下行为绝对禁止，一旦出现必须立刻重构修改：

- 尽量减少 any、谨慎使用 ts-ignore，合理处理类型报错，不做强制硬性限制
- 禁止超长文件、禁止臃肿大组件
- 禁止冗余重复代码，复用必须抽离
- 禁止写死密钥、禁止明文暴露接口地址
- 禁止随意安装无用依赖，安装前评估体积和稳定性

---

九、项目风格定位（MetaCode 专属）

- 产品定位：AI 代码助手、多模型聚合、Agent 智能体。
- UI 风格：极简风、高级干净、低饱和、简约元素视觉。
- 交互逻辑：流畅、轻量化。
- 代码思想：模块化、插件化、可扩展、方便后续接入 MCP、插件系统。

---

十、给 AI 的强制指令
你（Claude）必须：

1. 每次生成代码前自查是否违反本文档；
2. 违反规则自动重构，无需询问用户；
3. 优先保证代码干净、短小、可维护；
4. 永远贴合 MetaCode 极简产品风格；
5. 输出回答简洁直白，不要废话、不要过度客套。
