# MetaCode 产品能力升级文档（Agent\+MCP\+技能\+插件体系）给Claude Code专用开发文档

- **产品名称**：MetaCode

- **产品定位**：本地 AI 代码编辑器、AI 编程助手、轻量化 Cursor / Claude Code 本土化客户端

- **技术栈**：Electron \+ React \+ Vite \+ TypeScript \+ Node\.js

- **模型方式**：默认本地模型\(Ollama\)、支持自定义API模型

- **现有功能**：代码对话、AI聊天、代码高亮、文件树、设置中心、模型管理、个性化UI

- **开发约束**：不重构现有页面、不破坏原有逻辑、所有新增功能模块化嵌入

# 2\. 本次升级开发目标（核心需求）

本次需要给 MetaCode 完整增加一套行业标准 AI 架构：**Tool工具 → Skill技能 → Plugin插件 → MCP调度中心 → Agent智能决策**。

最终效果对标：**Claude Code、Cursor**

实现能力：

1. AI 自动判断是否调用工具（时间、联网、本地文件）

2. 可视化插件管理中心（安装/卸载/启停）

3. 技能工作流系统（可编辑、可预设代码工作流）

6. 内置系统时间、本地命令、文件读写工具；联网搜索通过 MCP 接入

# 3\. 架构设计（严格遵循此结构开发）

## 3\.1 层级顺序（自上而下）

用户对话 → UI聊天层 → **MCP调度中心** → Agent决策层 → 能力池\(Tool/Skill/Plugin\) → 模型层

## 3\.2 四大核心模块定义（必须严格区分）

### 3\.2\.1 Tool（基础工具层｜支持第三方导入/远程订阅）

最小不可拆分执行单元，分为**内置工具、第三方工具、远程工具**三类，统一MCP工具协议。纯执行逻辑，无业务流程，是整个生态最底层原子能力。

内置工具清单：

- **内置原生工具**：getCurrentTime、readFile、writeFile、terminalCommand，客户端自带，永久可用。

- **第三方导入工具**：支持本地导入 \.mctool 工具配置文件，支持手动编写JSON工具。

- **远程订阅工具**：支持订阅第三方公开工具源、云端工具市场、API工具集合。

- **插件携带工具**：外部插件自带私有工具，插件安装自动注册、卸载自动销毁。

#### 工具能力规范（开放生态）

- 协议兼容：OpenAI Function / Anthropic Tool / MetaCode MCP 通用格式。

- 权限管控：每个工具独立权限，文件/联网/终端单独授权。

- 沙箱隔离：第三方工具禁止直接访问本地敏感路径、密钥配置。

- 导入导出：支持导出单工具、批量导出工具包、分享给其他用户。

### 3\.2\.2 Skill（技能工作流层｜支持第三方技能市场、导入导出）

由一个或多个 Tool 组合编排而成的**可编程工作流**，面向业务场景。分为内置技能、第三方社区技能、用户自定义技能。支持可视化编排、导入导出、技能市场下载，对标 Claude Skill 能力。

预设技能：

- **内置预设技能**：智能代码调试、联网查文档、项目结构分析、代码优化。

- **第三方社区技能**：技能市场下载他人封装好的工作流，支持热度排序、分类筛选。

- **用户自定义技能**：可视化拖拽编排工具顺序、自定义触发条件、自定义入参。

- **插件绑定技能**：插件可内置专属技能，安装插件自动解锁对应工作流。

#### 技能文件规范

- 文件格式：\.mcskill 标准化技能文件，可本地保存、互相分享。

- 依赖检测：技能绑定工具、插件依赖，缺失依赖自动提示安装。

- 执行模式：串行执行、并行执行、条件分支执行。

- 版本管理：技能支持版本升级、覆盖更新、回滚历史版本。

### 3\.2\.3 Plugin（插件扩展层｜对标 Claude Code / CCswitch）

插件是 MetaCode 最高层级扩展能力包，用于打包分发 Tool、Skill、事件钩子、自定义配置。完全对标 Claude Code、CCswitch 插件生态，支持**内置插件、线上市场、本地导入**三种安装方式，实现商业化插件生态体系。插件为可插拔、隔离运行、权限可控，支持版本管理、依赖管理、自动更新。

插件标准结构（统一插件协议）：

- **基础元信息**：插件ID、名称、描述、图标、版本、作者、开源地址、许可证

- **能力绑定**：绑定Tool工具、绑定Skill技能、自定义钩子事件

- **依赖管理**：依赖其他插件、依赖最低内核版本

- **权限声明**：文件权限、联网权限、终端权限、模型调用权限

- **配置项**：用户可视化配置面板、默认配置、动态修改

#### 插件三大安装来源（核心功能，对标行业顶级生态）

1. **内置插件（原生内置、无需下载）**打包进客户端，开箱即用

2. 包含：时间插件、文件增强插件、Git插件；联网搜索通过 MCP 服务接入

3. 不可卸载，仅可开关禁用

4. **线上插件市场（Marketplace）**官方插件市场：云端索引插件列表，支持模糊搜索、分类筛选、热度排序

5. 第三方市场：支持自定义添加插件源（Git仓库、第三方JSON市场源）

6. 交互逻辑：一键安装、一键更新、后台静默下载、解压校验、依赖自动安装

7. 缓存机制：插件压缩包缓存、版本比对、增量更新

8. **本地手动导入（私有插件）**支持本地 \.mcplugin 私有插件包（加密压缩格式）

9. 支持直接选择本地插件文件夹导入

10. 用于个人私有插件、企业内部定制插件、未公开测试插件

#### 插件运行隔离机制（安全管控）

- 插件沙箱运行，禁止越权调用敏感系统接口

- 权限白名单：联网、读写、终端、剪贴板分开授权

- 插件崩溃隔离：单个插件报错不会导致主程序崩溃

- 加载优先级：内置插件 \&gt; 线上插件 \&gt; 本地导入插件

### 3\.2\.4 MCP（模型控制协议·调度核心）

MetaCode自研轻量MCP协议，作用：

- 统一管理所有工具、技能、插件

- 给模型注入工具描述、参数规范

- 拦截模型返回，识别工具调用指令

- 顺序执行、循环执行、多轮调用

- 权限拦截（禁止危险命令）

- **第三方协议兼容**：支持接入外部第三方 MCP 服务、第三方工具源、第三方技能包

##### 5、统一权限隔离（第三方安全管控）

- 所有第三方能力默认**权限冻结**，需要用户手动授权

- 联网、文件、终端、剪贴板全部单独管控

- 第三方插件/工具禁止读取本地敏感配置、密钥

- 所有第三方调用日志独立记录，方便排查风险

## 3\.3 Agent 智能决策逻辑

模仿 Claude Code 原生Agent：

1. 用户提问 → MCP预处理（注入当前时间、上下文、可用工具）

2. 模型自主判断：是否需要工具？需要哪一个？

3. MCP捕获模型tool_call

4. 后端Node层安全执行工具

5. 工具结果回传给模型

6. 模型整合结果输出自然语言回答
