本功能核心目的：

- 实现长期本地记忆，模型不会遗忘早期对话

# 2\. 整体架构（固定不可修改）

## 2\.1 架构流程图

用户新提问 → 获取第三方Embedding向量 → 本地向量库相似度检索 → 取出相关历史记忆 \+ 最近固定轮数上下文 → 拼接上下文发送给大模型 → 模型回答 → 本轮对话向量化存入本地

## 2\.2 三层架构强制划分

1. **Embedding 抽象层**：统一适配所有 OpenAI 格式第三方 Embedding 接口

2. **本地向量存储层**：基于 sqlite\-vec，纯本地文件向量数据库

3. **记忆拼接管理层**：控制上下文、过滤记忆、控制 Token 大小

# 5\. 详细功能设计

## 5\.1 Embedding 抽象层设计

必须封装统一接口，不可写死任何一家模型：

```typescript
interface EmbeddingProvider {
  getEmbedding(text: string): Promise<number[]>
}
```

实现类：

- OpenAIEmbedding（通用兼容）

- DeepSeekEmbedding

- MoonShotEmbedding（月之暗面）

规则：用户在哪家模型配置页填入密钥，自动切换对应 Embedding 渠道。

## 5\.2 本地向量数据库结构

数据库表固定三张：

1. **chat\_session**：会话表（会话ID、创建时间、会话名称）

2. **chat\_message**：消息表（归属会话、角色、内容、时间）

3. **chat\_vector**：向量表（消息ID、向量数据、文本摘要）

## 5\.3 RAG 检索逻辑（强制执行）

### 5\.3\.1 检索流程

1. 用户输入新问题

2. 调用当前第三方模型 Embedding 接口，生成问题向量

3. 本地向量库进行相似度计算（余弦相似度）

4. 筛选相似度最高 **4 条历史相关记录**

5. 拼接：系统提示 \+ 检索记忆 \+ 最近5轮对话 \+ 当前问题

### 5\.3\.2 相似度阈值

- 相似度 ≥ 0\.4：保留记忆

- 相似度 \&lt; 0\.4：判定无关，不塞入上下文（节省Token）

## 5\.4 记忆写入规则

- 每轮对话结束后，异步向量化存入本地数据库

- 禁止同步阻塞聊天请求

- 长文本自动切片：单条超过 1000 字符自动分段向量化
